Chart

What's new

  • Make legendItemClick cancellable
  • Add imageDataURL method for exporting to image (requires Canvas support)
  • Add renderAs setting to specify preferred rendering mode (svg/canvas/vml)
  • Pie chart string values should be parsed to float.
  • Default axis maximum value is now chosen to closely match series extremes
  • Make legendItemHover cancellable
  • Series values are now aggregated even if a date contains only a single point
  • missingValues now defaults to "interpolate" for line series (was "gap")
  • Add markers.rotation option
  • Chart Annotations
  • Add categoryField for all categorical series. Categories (including dates) can be unique across series.
  • Polar and radar charts
  • Add seconds for the baseUnit in the chart.
  • Series options can now be functions that are executed for each point.

What's fixed

  • Bar charts are broken when no categories are defined
  • valueAxis.narrowRange is ignored when series contain null values
  • "count" aggregate includes undefined and null values in its result
  • "avg" aggregate includes null and undefined values when calculating average value
  • "avg" aggregate returns NaN when a data contains no values (regression in latest builds)
  • Data points might not be included in aggregates if series contain multiple values per date
  • Bullet chart does not render when categoryAxis.justified is false
  • Categories are serialized even when the category axis is bound
  • setOptions doesn't extend original options and doesn't apply theme
  • Error when creating bubble series with negative size
  • Disabled item in the pie chart breaks legend item click/hover functionality.
  • If candlestick line color is empty the line should get default series color.
